# Спецификация TeleMT Manager API

Этот сервис управляет ключами для прокси-сервера TeleMT. Все запросы к API (кроме `/health`) требуют заголовок `X-API-Key`.

**Базовый URL:** `http://<ip-вашего-сервера>:8000`  
**Авторизация:** Заголовок `X-API-Key: super-secret-key-123` (настраивается в `.env`)

---

## Эндпоинты

### 1. Создание гостевого ключа

Создает анонимный ключ, который автоматически удалится через 24 часа.

- **URL:** `/keys/guest`
- **Метод:** `POST`
- **Ответ (200 OK):** Объект с данными ключа и ссылкой `proxy_url`.

### 2. Создание постоянного ключа

Создает бессрочный ключ для Telegram ID. Если у пользователя уже был ключ, он будет заменен.

- **URL:** `/keys/permanent`
- **Метод:** `POST`
- **Параметры (Query):** `tg_user_id` (integer)

### 3. Удаление ключа по Telegram ID

Принудительный отзыв доступа.

- **URL:** `/keys/telegram/{tg_user_id}`
- **Метод:** `DELETE`

### 4. Список всех ключей

- **URL:** `/keys/`
- **Метод:** `GET`

---

## Формат ссылки (proxy_url)

Сервис генерирует ссылки **MTProxy Fake TLS**:
`tg://proxy?server={HOST}&port={PORT}&secret=ee{SECRET}{DOMAIN_HEX}`
Префикс `ee` включает режим TLS, а в конце добавлен HEX домена для маскировки (например, `google.com`).
